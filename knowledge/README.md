## Cómo leer la verificación

Cada dato sensible lleva una etiqueta de confianza:

- ✅ **Verificado** — fuente autoritativa + fecha (Stanford HAI, doc oficial, ley publicada).
- 🟡 **Probable** — fuente reputada pero secundaria/agregador; confírmalo antes de uso de alto riesgo.
- 🔴 **Pendiente de verificación** — dato volátil (modelos, precios, rankings) o no confirmado. **No lo afirmes como hecho cerrado en una ponencia sin revisar la fuente primaria ese día.**

**Regla de oro:** los rankings de modelos, precios y "quién es #1" cambian cada semana.
Trátalos como **foto del momento** (con fecha), no como verdad permanente. Para datos
duros usa siempre la **fuente primaria** (model card, AI Index, ley, paper).

## Cómo usar este conocimiento para monetizar

1. **Autoridad:** publica 1 insight/semana usando `ai-commercial-narratives.md` + un dato de `ai-facts.md`.
2. **Formación:** arma clases/webinars con `ai-learning-roadmap-12w.md` + `ai-frameworks.md`.
3. **Consultoría:** diagnostica con `ai-smart-questions.md` + `ai-frameworks.md` (ROI, riesgos).
4. **Ponencias:** abre con una anécdota (`ai-anecdotes.md`), sostiene con datos (`ai-facts.md`), cierra con framework.
5. **Estar al día:** corre el `ai-weekly-radar-template.md` cada lunes (30–45 min).
