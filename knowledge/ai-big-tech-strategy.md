## 1. OpenAI
- **Qué es:** lab + producto líder en consumo (ChatGPT). Asociada a Microsoft.
- **Modelos:** familia GPT (GPT-5/5.5 🔴), reasoning integrado, Sora (video). 🔴 verificar versiones.
- **Producto:** ChatGPT (consumer + enterprise), GPTs, API, Codex, plataforma de agentes.
- **Distribución:** marca #1 en consumo + Azure + API masiva.
- **Monetización:** suscripción (Plus/Pro/Team/Enterprise) + API por uso.
- **Infraestructura:** Azure + acuerdos de cómputo a gran escala.
- **Open:** mayormente **closed** (con aperturas puntuales). 🔴
- **Fortaleza:** marca, producto, ecosistema de devs, velocidad de release.
- **Debilidad:** costo, dependencia de cómputo, presión competitiva en precio.
- **Riesgo:** commoditización del modelo; gobernanza/seguridad bajo lupa.
- **Oportunidad:** construir productos verticales sobre su API.
- **Aprender:** **distribución + producto** ganan, no solo el modelo.
- **Ponencia:** "ChatGPT no inventó el LLM; inventó que todos lo usaran."
- **Contenido:** "Qué copiarle a OpenAI sobre time-to-market."

## 2. Anthropic
- **Qué es:** lab enfocado en **seguridad/alineamiento**; creador de Claude.
- **Modelos:** Claude (Opus/Sonnet/Haiku; Opus 4.8, Fable 5 🔴). Fuerte en coding, escritura, docs largos, agentes; **MCP**.
- **Producto:** Claude (web/app), Claude Code, API, Claude en nubes (AWS Bedrock, GCP).
- **Distribución:** API + partnerships cloud (AWS/Google) + enterprise.
- **Monetización:** suscripción + API; foco enterprise/developer.
- **Infraestructura:** cómputo vía AWS/Google (inversiones cruzadas).
- **Open:** **closed**, pero abrió **MCP** como estándar abierto.
- **Fortaleza:** calidad en código/razonamiento/escritura; confianza; MCP.
- **Debilidad:** menor presencia consumer que OpenAI/Google.
- **Riesgo:** competir en distribución masiva.
- **Oportunidad:** soluciones enterprise donde importa confianza/trazabilidad (legal, AEC, finanzas).
- **Aprender:** **vender confianza** (seguridad) como diferenciador.
- **Ponencia:** "Claude vendió confianza, no solo capacidad."
- **Contenido:** "Constitutional AI explicado para gerentes."

## 3. Google DeepMind
- **Qué es:** fusión Google Brain + DeepMind; modelos Gemini + investigación frontier (AlphaFold, etc.).
- **Modelos:** Gemini (3.x 🔴), multimodal nativo, contexto largo; modelos científicos.
- **Producto:** Gemini en Workspace, Búsqueda, Android, Vertex AI (enterprise).
- **Distribución:** **imbatible** — miles de millones de usuarios ya en sus apps.
- **Monetización:** Workspace/Cloud + publicidad + API (Vertex/AI Studio).
- **Infraestructura:** **TPUs propias** (ventaja de costo y soberanía de cómputo).
- **Open:** **híbrido** (Gemini closed + Gemma open-weight).
- **Fortaleza:** distribución + infra propia + investigación + multimodal/contexto.
- **Debilidad:** ejecución de producto a veces lenta; rutas consumer/dev/cloud confusas.
- **Riesgo:** disrupción de su negocio de búsqueda por la propia IA.
- **Oportunidad:** equipos que viven en Google Workspace.
- **Aprender:** **distribución + infra propia** es foso defensivo.
- **Ponencia:** "Google no necesita que descargues su IA: ya vive en tu Gmail."
- **Contenido:** "Por qué las TPUs son el arma secreta de Google."

## 4. Microsoft
- **Qué es:** plataforma + distribución enterprise; socio histórico de OpenAI; modelos propios (Phi) y MAI.
- **Modelos:** Phi (small/efficient, open), modelos propios + integración de terceros (OpenAI, Anthropic).
- **Producto:** Microsoft 365 Copilot, GitHub Copilot, Copilot Studio, Azure AI Foundry.
- **Distribución:** **Office + Windows + Azure** = base instalada gigante.
- **Monetización:** licencias Copilot por asiento + Azure por consumo.
- **Infraestructura:** Azure (hyperscaler) + cómputo a escala.
- **Open:** **híbrido** (Phi open; plataforma multi-modelo).
- **Fortaleza:** distribución enterprise, plataforma multi-modelo, GitHub.
- **Debilidad:** depender de socios para frontera; complejidad de licencias.
- **Riesgo:** tensión con OpenAI; canibalización de productos.
- **Oportunidad:** empresas Microsoft que adoptan Copilot por asiento.
- **Aprender:** **ser la plataforma** (no solo el modelo) captura valor.
- **Ponencia:** "Microsoft no apuesta a un modelo; apuesta a ser el lugar donde corren todos."
- **Contenido:** "Copilot por asiento: cómo calcular el ROI real."

## 5. Meta AI
- **Qué es:** impulsor #1 del **open-weight** (Llama); IA en sus apps (WhatsApp, IG, FB).
- **Modelos:** Llama (open-weight). 🔴 verificar versión vigente.
- **Producto:** Meta AI en sus apps + Llama para devs/empresas.
- **Distribución:** miles de millones vía sus apps; ecosistema open enorme.
- **Monetización:** indirecta (publicidad, engagement); open como estrategia.
- **Infraestructura:** capex masivo en GPUs/data centers.
- **Open:** **open-weight** (bandera del movimiento).
- **Fortaleza:** ecosistema abierto, distribución social, talento.
- **Debilidad:** monetización directa de IA poco clara; ritmo frontier variable.
- **Riesgo:** que labs chinos lideren el open-weight que Meta abrió.
- **Oportunidad:** pymes/devs que self-hostean Llama por costo/privacidad.
- **Aprender:** **abrir** commoditiza al rival y construye ecosistema.
- **Ponencia:** "Meta convirtió el modelo en infraestructura pública."
- **Contenido:** "Open-weight para pymes: cuándo conviene self-host."

## 6. Amazon / AWS
- **Qué es:** la **nube** de la IA empresarial; modelos propios (Nova) + Bedrock (multi-modelo). Invierte en Anthropic.
- **Modelos:** Amazon Nova (familia propia) + acceso a Anthropic/Meta/Mistral/etc. vía Bedrock.
- **Producto:** Bedrock (modelos como servicio), Q (asistente), SageMaker.
- **Distribución:** base AWS (líder cloud) + marketplace.
- **Monetización:** consumo cloud + inferencia gestionada.
- **Infraestructura:** AWS + chips propios (Trainium/Inferentia).
- **Open:** **agnóstico** (vende acceso a todos).
- **Fortaleza:** ser la **plataforma neutral** + chips propios + clientes enterprise.
- **Debilidad:** modelos propios menos visibles en frontera.
- **Riesgo:** que la batalla la ganen quienes tienen consumer/marca.
- **Oportunidad:** empresas que ya están en AWS y quieren multi-modelo gobernado.
- **Aprender:** en una fiebre del oro, **vende picos y palas** (infra).
- **Ponencia:** "Amazon no necesita el mejor modelo: necesita que todos corran en su nube."
- **Contenido:** "Bedrock vs. elegir un solo proveedor: pros y contras."

## 7. NVIDIA
- **Qué es:** el **proveedor de cómputo** (GPUs) que habilita toda la industria; además software (CUDA) y modelos (Nemotron).
- **Modelos:** Nemotron (open) + plataforma (NIM, NeMo) — su core es **hardware + CUDA**.
- **Producto:** GPUs (data center), CUDA, frameworks, microservicios de inferencia.
- **Distribución:** estándar de facto en entrenamiento/inferencia.
- **Monetización:** venta de GPUs + software/plataforma.
- **Infraestructura:** **es** la infraestructura.
- **Open:** mixto (modelos open + ecosistema propietario CUDA).
- **Fortaleza:** **moat de CUDA** + liderazgo absoluto en GPUs + demanda insaciable.
- **Debilidad:** concentración/ciclos de demanda; competencia en chips (TPU, Trainium, Huawei).
- **Riesgo:** que hyperscalers escalen sus chips propios; geopolítica de exportación.
- **Oportunidad:** todo lo que sea infra/edge AI corre sobre su stack.
- **Aprender:** **el cuello de botella físico** captura un valor enorme.
- **Ponencia:** "En la fiebre del oro de la IA, NVIDIA vende las palas — y casi todas son suyas."
- **Contenido:** "Por qué CUDA es el verdadero foso de NVIDIA."

## 8. xAI
- **Qué es:** lab de Elon Musk; modelo Grok integrado a X; cómputo a gran escala (Colossus).
- **Modelos:** Grok (acceso a info en tiempo real de X). 🔴 versión.
- **Producto:** Grok en X + API; posicionamiento "anti-censura/realtime".
- **Distribución:** integración con X (red social) + API.
- **Monetización:** suscripción X Premium + API.
- **Infraestructura:** clúster propio de gran escala.
- **Open:** mixto (ha liberado versiones).
- **Fortaleza:** datos en tiempo real (X), capital, cómputo, narrativa.
- **Debilidad:** gobernanza/seguridad cuestionadas; sesgo de fuente (X).
- **Riesgo:** reputación/moderación; competencia frontier.
- **Oportunidad:** casos de "pulso en tiempo real" y tendencias.
- **Aprender:** **datos propios en tiempo real** son diferenciador difícil de copiar.
- **Ponencia:** "Grok apuesta a un dato que nadie más tiene: lo que pasa ahora mismo."
- **Contenido:** "Realtime AI: cuándo importa el dato del minuto."

## 9. Mistral AI
- **Qué es:** campeón europeo; modelos eficientes, fuerte en **open-weight**.
- **Modelos:** familia Mistral/Mixtral (MoE), modelos open + comerciales. 🔴 versión.
- **Producto:** API (La Plateforme), Le Chat, modelos open para self-host.
- **Distribución:** API + nubes + soberanía europea.
- **Monetización:** API + licencias enterprise + soporte.
- **Infraestructura:** Europa (soberanía de datos).
- **Open:** **open-weight** fuerte + comercial.
- **Fortaleza:** eficiencia, soberanía UE, open, talento.
- **Debilidad:** escala/capital menores que US/China.
- **Riesgo:** quedar entre US (capital) y China (precio).
- **Oportunidad:** clientes UE con requisitos de soberanía/privacidad.
- **Aprender:** **eficiencia + soberanía** como nicho defendible.
- **Ponencia:** "Europa juega la carta de la soberanía y la eficiencia."
- **Contenido:** "Soberanía de datos: por qué algunas empresas eligen Mistral."

## 10. Cohere
- **Qué es:** lab **enterprise-first** (no consumer); foco en RAG, embeddings y despliegue privado.
- **Modelos:** Command (generación) + Embed/Rerank (búsqueda) — familia enterprise.
- **Producto:** modelos para empresa, RAG, on-prem/VPC, multilingüe.
- **Distribución:** ventas enterprise + nubes.
- **Monetización:** licencias enterprise + API.
- **Infraestructura:** agnóstica (despliegue en la nube del cliente).
- **Open:** mixto (algunos pesos abiertos).
- **Fortaleza:** **RAG/embeddings**, privacidad, multilingüe, enterprise.
- **Debilidad:** poca marca consumer; frontera no es su pelea.
- **Riesgo:** competir con hyperscalers en enterprise.
- **Oportunidad:** empresas que necesitan **RAG privado** y datos en su VPC.
- **Aprender:** **especializarse** (no pelear la frontera consumer) es viable.
- **Ponencia:** "No todos pelean por el mejor chatbot; algunos ganan en el dato privado."
- **Contenido:** "Embeddings + rerank: el motor invisible del RAG."

## 11. IBM
- **Qué es:** IA empresarial **gobernada**; modelos Granite (open) + plataforma watsonx; foco en regulados.
- **Modelos:** Granite (open-weight, enterprise, eficientes).
- **Producto:** watsonx (modelos + datos + gobernanza), consultoría.
- **Distribución:** clientes enterprise/regulados + servicios.
- **Monetización:** plataforma + consultoría + licencias.
- **Infraestructura:** híbrida/on-prem + nube.
- **Open:** **open-weight** (Granite) con licencias claras.
- **Fortaleza:** gobernanza, confianza en regulados, consultoría, on-prem.
- **Debilidad:** percepción de "lento"; no es frontier.
- **Riesgo:** quedar como integrador más que innovador.
- **Oportunidad:** banca/seguros/sector público que exigen gobernanza.
- **Aprender:** **gobernanza + servicios** monetizan en regulados.
- **Ponencia:** "En sectores regulados, gana la IA auditable, no la más llamativa."
- **Contenido:** "IA gobernada: checklist para sectores regulados."

## 12. AI21 Labs
- **Qué es:** lab israelí; modelos Jamba (arquitectura híbrida SSM/Transformer) y soluciones enterprise.
- **Modelos:** Jamba (contexto largo eficiente) + productos de tareas (lectura/escritura).
- **Producto:** API + soluciones empresariales (lectura, resumen, agentes).
- **Distribución:** API + enterprise + nubes.
- **Monetización:** API + licencias.
- **Infraestructura:** agnóstica.
- **Open:** mixto (Jamba con pesos abiertos en versiones).
- **Fortaleza:** **arquitecturas eficientes** (híbridas) + contexto largo.
- **Debilidad:** escala/marca menores.
- **Riesgo:** nicho estrecho frente a gigantes.
- **Oportunidad:** casos de contexto largo con costo controlado.
- **Aprender:** **innovar en arquitectura** abre espacio frente a los grandes.
- **Ponencia:** "No toda la innovación es escala; algunos ganan con mejor arquitectura."
- **Contenido:** "Más allá del Transformer: arquitecturas híbridas (SSM)."
